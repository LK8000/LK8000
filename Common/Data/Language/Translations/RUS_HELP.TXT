@121

@122

Захламление на карте может перекрыть всю полезную информацию. Этот параметр полезен для ускорения работы карты и экрана, когда у вас имеется огромное количество обозначений топологии и путевых точек, а процессор устройства медленный. Как правило, вы можете оставить это значение по умолчанию - 70

@123
Определяет, как надписи отображаются ко всем точкам:
 [Названия] отображается полное название каждой точки.
 [Номера] отображается только номера каждой точки.
 [Нет] Названия точек не отображаются.
 [Название в задаче] отображаются только названия для точек,  которые находятся в задаче, а также домашний аэродром.
 [Первые три] отображаются первые 3 буквы в названии точек.
 [Первые пять] отображаются первые 5 букв в названии точек.

@124

@125
Определяет, как экран и планер вращаются относительно друг друга:
 [На север] перемещаемая карта на экране всегда будет ориентирована  с севера на юг, а значок планера будет менять положение. 
 [Умный север] Так же как и На север, но значок планера позиционируется автоматически, освобождая место впереди для карты. Этот механизм работает только в пейзажном режиме. Для портретного режима, он идентичен На север. В режиме Умный север не возможно вручную изменять положение планера кнопками Меню.
 [По треку] перемещение карты на экране будет изменяться таким образом, что трек планера всегда ориентирован вверх.
 [Поворачивать север] Трек ориентирован вверх на переходах, север вверху в спирали.
 [Поворачивать цель] Трек ориентирован вверх на переходах, следующая точка вверху в спирали.
 [Север/трек] Север ориентирован вверх на переходах, трек планера ориентирован вверх в спирали.

@126
Когда масштаб карты больше или равен этому значению,  ориентация карты изменится в режим На север.
Ниже этого значения, настройки ориентации будут использованы нормально.
Настройка высокого значения приведет к тому, что этот вариант никогда не будет использоваться. По умолчанию 10.

@127
Автомасштаб - изменение масштаба во время полета таким образом, что следующая точка постоянно находится в поле зрения на краю экрана. После взятия точки, в поле зрения попадает следующая путевая точка.
Ручное изменение масштаба приведет к отключению функции автоматического масштабирования.

@129

@130
Если включен этот режим, то карта будет увеличиваться автоматически в спирали, и автоматически уменьшаться при прекращении вращения.

@131
Значки точек имеют такой высокий приоритет над значками топологии, что точки на карте могут полностью перекрывать значки топологии. Здесь вы можете определить, сколько вы хотите отображать точек, во избежание захламления карты и подтормаживания процессов LK8000.
 ОТКЛЮЧИТЬ - покажет все путевые точки на карте, даже перекрывающие друг друга.
 МАЛЫЙ - не будет показывать перекрывающие друг друга точки,  но заполнит карту по максимуму.
 СРЕДНИЙ и БОЛЬШОЙ - оставляет больше свободного места между значками точек, освобождая его для значков топологии. 
 ОЧЕНЬ БОЛЬШОЙ - будет сильно отделять значки точек, почти отдавая приоритет значкам топологии.

@132
 ОТКЛЮЧИТЬ - покажет информацию для всех достижимых посадок, которые видны на карте.
 МАЛЫЙ - не будет показывать посадки, если есть лучшие доступные аэропорты.
 БОЛЬШОЙ - будет пытаться не показывать аэропорты близко к другим лучшим аэродромам.

@133

@134
Отображает безопасную высоту для посадки только на посадочные точки, или также все маршрутные точки. Если только посадочные точки, то высота прибытия над маршрутной точкой не будет рассматриваться как безопасная, а точки, как запасные посадки.

 [Только посадки] Применяет дополнительную высоту для безопасного приземления планера только для посадочных точек. Данное значение стоит по умолчанию.
 [Посадки и точки] Применяет дополнительную высоту для безопасного приземления планера для всех маршрутных точек 

@135

Рекомендованное значение 50.

@136

@137

Это не будет происходить ниже критической высоты, чтобы не
беспокоить вас.

@139
Эта функция позволяет включать или выключать алгоритм автоматического определения ветра.
Когда алгоритм выключен, пилот самостоятельно отвечает за оценку ветра.
 [Спираль] Требуется только источник GPS данных.
 [ZigZag] Требуется интеллектуальный Вариометр с выходом воздушной скорости.
 [Оба] Используется ZigZag и спираль.

@140
Это воздушная скорость в прямолинейном полете. Она используется до того, как произойдет расчет истинного ветра.
Постарайтесь выдерживать четкий курс, глядя на ваш компас, и пролетая поочередно на север, восток, юг и запад и, выдерживать эту воздушную скорость на анемометре на несколько секунд. 
Затем нажмите специальную кнопку Расчет Ветра в главном меню. Если у вас нет зонда воздушной скорости, нужно указать LK8000 какова ваша приборная скорость по анемометру. LK предполагает, что вы будете использовать это значение. 
Если у вас есть зонд воздушной скорости полета, это значение не используется, а ветер автоматически рассчитывается по измеренной воздушной скорости. 
Минимальная скорость - 5 узлов или 10 км/ч, по умолчанию  - 100 км/ч или 54 узла.

@141
Для расчета ветра во время прямолинейного полета вы должны в течении нескольких секунд удерживать без изменения скорость и курс. Решите, на сколько секунд вы сможете удерживать стационарные значения. По умолчанию это значение 10 секунд.
Для более длительного периода времени трудно выполнить необходимые условия. Напротив, в более короткий период времени выполнить их гораздо легче, но при этом расчеты могут быть менее точными.
Для этого параметра существует допустимая погрешность 30%.

@142
Этот параметр определяет, какой автоматический алгоритм MacCready будет использоваться.
 [Финишный долет] регулирует MacCready для быстрого финиширования. Для OLC Sprint задач MacCready будет регулировать преодоление наибольшего расстояния в оставшийся период времени и прилет к финишу с наличием высоты.
 [Средний подъем] усредненное значение MacCready из значений всех потоков.
 [Оба] использует средние значения MacCready во время выполнения задачи, и быстрое финиширование на финишном долете.
 [Эквивалент Mc] Значения MacCready рассчитывается на средней скорости. Смотрите руководство

@143
Аэродинамическое качество всегда рассчитывается в реальном времени. Здесь вы можете решить, за сколько секунд полета этот расчет должно быть сделан и усреднен. Реальный путь, пройденный с каждой секундой в этот период времени, делится на образовавшуюся разницу высоты.
Например, если вы установили это значение в 2 минуты, то вам будет показано усредненное значение АД качества за предыдущие 2 минуты полета, а не текущее на настоящий момент. Как правило, эти значения могут совпадать, но с учетом забросов и пр. могут немного различаться
Обычно для планеров хорошее значение 90-120 секунд, а для  парапланов 15 секунд.
Более низкие значения будут давать в результате, в значительной степени, показания практически приближенные к мгновенным, в то время как более высокие значения будут давать более реальные значения АД качества на переходе. 
Другое аналогичное программное обеспечение и дорогие летные приборы используют по умолчанию для вычисления этого показателя значение 2 минуты.

@158
Коррекция абсолютной высоты (высоты уровня моря, Геоид)  применяется только к GPS  высоте, а не к барометрической высоте. Если у вас есть источник данных барометрической высоты, и вы в настройках выбрали его использование, этот параметр не имеет значения. 
Если у вас нет барометрической высоты, или она недоступна, то вы должны убедиться, компенсировал ли уже источник GPS данных разницу в абсолютной высоте или еще нет.
Если показания высоты, когда вы находитесь на земле постоянно составляют от 30 до 50 м., но вы уверены, что это не так, то вы должны самостоятельно решить, либо включить, либо отключить эту опцию. Если есть сомнения, лучше оставить эту опцию включенной.
Некоторые модели GPS уже компенсируют расхождения абсолютной высоты.
При использовании с планерным тренажером-симулятором "Кондор"  необходимость в этом параметре отсутствует.
По умолчанию режим включен.

@159

Перед изменением этого значения, убедитесь, что смещение абсолютной высоты применяется правильно. Значение по умолчанию равно 0.
Это значение сохраняется в реестре!

По умолчанию установлен режим Норма. Для ПК этот выбор ни на что не влияет.

@303
Эта настройка только для барометрической высоты, если она доступна! Если у вас нет барометрической высоты, эта конфигурация ни на что не влияет!
Это атмосферное давление в данном районе для калибровки барометрического высотомера. Некоторые внешние Вариометры будут автоматически устанавливать ее, другие требуют ваших действий. После изменения QNH будет соответственно установлена новая барометрическая высота. При изменении высоты в следующем окне, QNH также будет соответственно установлена.
Вы обычно устанавливаете правильную высоту на земле, и множество новых QNH во время полета, если метеослужба в зоне полетов предоставит вам информацию о новых данных QNH.
Подчеркиваем, речь идет о барометрической высоте, и если у вас нет барометрического датчика высоты, данные которого предаются на LK, то программа с этими данными ничего не будет делать.
Стандартное давление на высоте QNE является 1013,2. Это значение выставлено по умолчанию. Если вы не понимаете, о чем идет речь, не меняйте QNH.
QNH (произносится как Кю-Эн-Эйч). Давление, приведенное к уровню моря. При установке этого давления на высотомере, прибор будет показывать высоту полета воздушного судна относительно уровня Балтийского моря (этот уровень является мировым эталоном). Для того, чтобы определить реальную высоту полета воздушного судна над земной поверхностью, используя этот тип давления, потребуется знать лишь превышение рельефа земли над уровнем моря. Эти цифры печатают на любой топографической карте. Т.е. вычитая из высоты на высотомере, превышение рельефа местности над уровнем моря, и получают истинную высоту полета над землей. Однако, поскольку рельеф под самолетом в процессе полета постоянно меняется, значение истинной высоты полета также изменяется, и при пилотировании обычно не используется.

@304
Здесь меняется барометрическая высота. У вас должен быть источник данных барометрической высоты, которая передается в LK, в противном случае это значение всегда будет равно нулю.
Изменение барометрической высоты соответственно автоматически изменит и QNH.
Если вы хотите установить QNE высоту, отрегулируйте QNH на 1013,2.
QNE (произносится Кю-Эн-И). Стандартное давление. На высотомере устанавливается заранее определенное значение: 760 мм рт.ст., или 1013 гПа, или 29.92 д рт.ст. При установке такого давления на высотомере, отображаемая прибором высота никоим образом не будет привязана ни к реальной высоте полета, ни к изменяющемуся атмосферному давлению. Смысл установки стандартного давления в том, что при полетах экипажу не придется каждый раз переставлять давление при переходе из одной погодной зоны в другую. Но при этом у всех воздушных судов, установивших стандартное давление (QNE), будет единая точка отсчета высоты, и, соответственно при полете на разных высотах будет обеспечиваться тот самый вертикальный интервал, ради которого все это и делается.

@650
[Местность]
Отображает на карте местности данные о погоде.

@651
[Wstar]
Average dry thermal updraft strength near mid-BL height.  
Subtract glider descent rate to get average vario reading for cloudless thermals.
Updraft strengths will be stronger than this forecast if convective clouds are present,
since cloud condensation adds buoyancy aloft (i.e. this negects cloudsuck).  
W* depends upon both the surface heating and the BL depth.

@652
[BL wind spd]
The speed and direction of the vector-averaged wind in the BL.  
This prediction can be misleading if there is a large change in wind direction through the BL.

@653
[H bl]
Height of the top of the mixing layer, which for thermal convection is the average top of a dry thermal.
Over flat terrain, maximum thermalling heights will be lower due to the glider descent rate and other factors.
In the presence of clouds (which release additional buoyancy aloft, creating cloudsuck) the updraft top will 
be above this forecast, but the maximum thermalling height will then be limited by the cloud base.
Further, when the mixing results from shear turbulence rather than thermal mixing this parameter is not useful for glider flying. 

@654
[dwcrit]
This parameter estimates the height above ground at which the average dry updraft strength drops below 225 fpm 
and is expected to give better quantitative numbers for the maximum cloudless thermalling height than the BL Top height,
especially when mixing results from vertical wind shear rather than thermals. 
(Note: the present assumptions tend to underpredict the max. thermalling height for dry consitions.)
In the presence of clouds the maximum thermalling height may instead be limited by the cloud base.
Being for "dry" thermals, this parameter omits the effect of "cloudsuck".

@655
[bl cloudpct]
This parameter provides an additional means of evaluating the formation of clouds within the BL and might be used either 
in conjunction with or instead of the other cloud prediction parameters.  
It assumes a very simple relationship between cloud cover percentage and the maximum relative humidity within the BL.
The cloud base height is not predicted, but is expected to be below the BL Top height.

@656
[Sfc temp]
The temperature at a height of 2m above ground level.  This can be compared to observed surface temperatures as an indication of model simulation accuracy; 
e.g. if observed surface temperatures are significantly below those forecast, then soaring conditions will be poorer than forecast.

@657
[hwcrit]
This parameter estimates the height at which the average dry updraft strength drops below 225 fpm and is expected 
to give better quantitative numbers for the maximum cloudless thermalling height than the BL Top height, 
especially when mixing results from vertical wind shear rather than thermals. 
(Note: the present assumptions tend to underpredict the max. thermalling height for dry consitions.) 
In the presence of clouds the maximum thermalling height may instead be limited by the cloud base.  
Being for "dry" thermals, this parameter omits the effect of "cloudsuck".

@658
[wblmaxmin]
Maximum grid-area-averaged extensive upward or downward motion within the BL as created by horizontal wind convergence. 
Positive convergence is associated with local small-scale convergence lines.
Negative convergence (divergence) produces subsiding vertical motion, creating low-level inversions which limit thermalling heights.

Между этими двумя крайностями лежит оптимальная скорость, при которой продвижение по маршруту максимально быстрое. Она рассчитывается, используя данные мгновенного MacCready и вариометра. На переходах эта скорость рассчитывается для поддержания высоты. При финишном долете эта скорость рассчитывается для снижения. 
В режиме набора высоты переключается на скорость минимального снижения с учетом текущего коэффициента перегрузки (если акселерометр подключен).

[Прозрачная] Зона ВП будет залита сплошным цветом, которому можно установить уровень прозрачности в следующем пункте меню.
Прозрачность может быть не доступна на некоторых устройствах.
Прорисовка границ воздушного пространства по шаблону может только замедлить устройств с высоким разрешением экрана или медленный процессор. Используйте с осторожностью!

@1201
[Прозрачность]
Определяет уровень прозрачности сплошной заливки цветом. Прозрачный - 0%, непрозрачный - 100%.

@1202
[Прозрачность поля данных]
Определяет уровень прозрачности нижнего поля данных.
Прозрачный - 0%, непрозрачный - 100%.
Обычно поле данных черное с белым текстом, или негатив - белый с черным текстом.
При значении <30% поле будет выглядеть очень прозрачным, и цвет текста будет инвертирован.
Некоторые комбинации прозрачности и инвертированного цвета не могут быть пригодны для навигации, но есть много комбинаций для выбора.
Использование 0% или 100% (полная прозрачность или непрозрачность) немного ускорит работу программы.
По умолчанию составляет 60%.
ВНИМАНИЕ: некоторые устройства не поддерживают прозрачность. 
В этом случае эта настройка ни на что не влияет.

@1203
[Сглаживание шрифтов]
Сглаживание шрифтов сделает лучше отображение символов на экране.
[Нет] отключает сглаживание шрифтов
[Обычный] будет использовать качество по умолчанию
[Сглаживание] нормальное качество
[Чистый шрифт] улучшенное качество
Обратите внимание, что на некоторых устройствах Сглаживание не работает.
О некоторых ПНА в портретном режиме, а на PPC2002, сглаживание будет работать лучше.
В противном случае, по умолчанию LK8000 использует Чистый шрифт.
Вы должны выйти и перезапустить LK8000, чтобы увидеть эффект от изменений.

@1204
[Режим шкалы варио]
Выберите режим отображения шкалы вариометра слева на экране, если она включена.
В потоке отображается только варио.
На переходе может быть нетто варио или Sрeed to Fly.
Если выбран Sрeed to Fly, на экране отображается отметка рекомендованной воздушной скорости, и вы должны увеличить или уменьшить свою скорость, чтобы индикатор достиг этой отметки.
Отметка выводится с буквами:
NT для нетто
SF для Sрeed to Fly

@1205

Каждый режим имеет уникальные особенности, пожалуйста, посетите наш веб-сайт или ознакомьтесь с последним руководством, чтобы понять различия.
После изменения режима, необходимо перезапустить ЛК, чтобы изменения вступили в силу.

@1208
[Тип ЛА]
Эта настройка содержит набор характеристик самолетов, планеров и парапланов различных классов.

@1209
[Время повтора сообщений ВП]
Это временной интервал, с которым сообщения о вторжении в воздушное пространство будут повторяться, если вы не подтвердили, что сообщение вами прочитано.
Если вы влетели в запрещенную зону воздушного пространства, и проигнорировали предупреждающее сообщение, то по истечении этого времени сообщение всплывет вновь.

@1210
[Вертикальное расстояние до ВП]
Если вертикальное расстояние до воздушного пространства сверху или снизу падает ниже этого предела, то всплывет желтое диалоговое окно предупреждения.
Это также влияет на обозначение предупреждения воздушного пространства, рисуемое на карте.
Если расстояние по вертикали (перепад высот) до нижней или верхней границы воздушного пространства ниже этого предела, значок предупреждения отображается непосредственно над символом планера.

@1211
[Интервал показа диалогового окна ВП]
Если это время истекает, диалоговое окно с предупреждением о воздушном пространстве закрывается автоматически, без нажатия на кнопку закрытия.
В диалоговом окне с предупреждением о воздушном пространстве вы можете увидеть титры обратного отсчета уменьшения счетчика на кнопке, показывающем оставшееся время до закрытия окна.

@1212
[Предупреждающие надписи ВП на карте]
Если выбрать этот пункт, то на предупреждающих обозначениях воздушного пространства, отображаемых на карте, будут указаны названия зон воздушного пространства и расстояние по вертикали до их границ.

@1213
[Порог автомасштаба]
При использовании Автомасштаба в режиме Параплан, Автомасштаб начинает работать, только если расстояние до следующей точки меньше, чем этот предел.
Например, если Вы выбираете 2 км, то когда вы будете в 2 км от следующей точки Автомасштаб начнет работать.
Он будет увеличивать карту по мере приближения к точке. После достижения путевой точки будет восстановлен предыдущий Масштаб перехода. И как только вы
достигает 2 км к следующей точке Автомасштаб опять начнет работать.

@1220
[Настройки нижнего поля данных]
Включение или отключение блоков нижней панели полей данных, при их перелистывании.
Если все блоки отключены, НАВ1 всегда отображается автоматически.
В режиме термического потока информационная панель ТРМ0 всегда отображается.

@1221
[Настройки перелистываемых Инфо-Страниц]
Включение или отключение каждой страницы в соответствии с их последовательностью перелистывания.
Если все элементы страницы будут отключены, то эта страница при перелистывании будет пропущена.
Такая страница по-прежнему будет доступна при помощи специальной пользовательской кнопки.
Внимание: при выборе конфигурации для таких летательных аппаратов, как парапланы и дельтапланы, произойдет отключение страницы 1,5 Соревнования, и также отключатся все расчеты для соревнований. Если вы хотите, чтоб соревнования рассчитывались, вам необходимо активировать страницу 1.5.
Если вы используете режим Планер, то вы также можете отключить страницу 1,5, т.к. расчеты будут производиться в любом случае.

@1250
Управление фильтрами воздушного пространства для отображения и предупреждения. Кнопка фильтра воздушного пространства также позволяет фильтровать отображение на экране и предупреждения независимо для каждого класса воздушного пространства.
[Все] Отображается все.
[В диапазоне] Только воздушное пространство в диапазоне высот.
[Авто] Все в пределах высоты летательного аппарата.
[Все ниже] Все воздушное пространство под планером и в пределах установленного превышения высоты над зоной.

@1251
Превышение высоты, которое используется для контроля воздушного пространства на движущейся карте. Используется только в режимах "Авто" и "Все ниже". Для режима "Авто" определяется высота зон воздушного пространства, которые должны быть показаны над и под планером.
Режим "Все ниже" покажет воздушное пространство до заданной высоты над планером, и все воздушное пространство под ним.

@1252
Даже если вы подтвердили, что приняли сообщение о пересечении границы воздушного пространства, по истечении этого времени режим предупреждения вновь активируется.
Вы можете быть предупреждены еще раз во время полета домой и повторного пересечения воздушного пространства, если вы, например, пересекли его утром!

@1253
[Сигнал максимальной высоты] Сигнал прозвучит каждый раз, когда эта высота будет достигнута и пересечена.
Сигнал не будет повторяться в течение следующих 60 секунд.
Через 1 минуту, если высота была снижена, чем установленная максимальная высота, а затем вновь становится выше, сигнал прозвучит вновь.
Для отключения этого сигнала, установите его в 0.
Высота, используемая для данного сигнала, будет той, которую вы выбрали в меню Настройки 5 "Использование барометрической высоты".
Сигнализация, установленная на той же высоте, не будет иметь никакого эффекта.
Пилоты должны быть осторожны, чтобы не настроить сигналы ниже высоты безопасности во время критических этапов полета.
Каждый сигнал может быть повторен не более 30 раз. После 30 повторов сигнализация автоматически самостоятельно отключится для текущего полета.
Визуальное сообщение сигнализации автоматически исчезает через 12 секунд.
Каждый сигнал имеет пользовательский звук, который может быть изменен в папке _System _Sounds.
Файл называется LK_ALARM_ALTx.WAV
По умолчанию: 0 (нет сигнала).

@1256
[Лучший глайд] Когда функция Лучший глайд используется, LK рассчитает прогнозируемую высоту прилета, при условии изменения текущей скорости, вплоть до скорости оптимальной для лучшего глайда.
Например, если вы летите на скорости 60 км.ч. без использования балласта, скорость Лучшего глайда будет рассчитываться, как если бы вы замедлили полет до, например, 36 км.ч., и при этом вы, конечно, прилетите с дополнительным запасом высоты.
Эта скорость будет использоваться для расчета всех высот прибытия. 
По умолчанию эта функция отключена по соображениям безопасности.
Если вы сомневаетесь, оставьте этот параметр выключенным.

@1257
[Качество]
Здесь указывается чистота планера. Установите 100%, если ваш планер идеально вымыт и на нем нет налипших насекомых и другого налипшего мусора. Значение 50% увеличит скорость снижения планера в два раза выше стандартного. 
Это значение по умолчанию используется Качество из меню Основные настройки, загруженные при запуске. Обычно Вы должны оставить это значение здесь 100%.
